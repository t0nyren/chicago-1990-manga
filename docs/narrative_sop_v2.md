# Narrative SOP v2 · 叙事手册

> Tony 2026-06-07 反馈：第一话 v1 看完"不是特别看得懂，背景交代太少，也很难通过对白知道他们在做什么"。
> 这份 SOP 是 v2 制作前提。从鬼灭之刃这类成熟连载学的叙事技法，全部应用到芝加哥 1990 漫画化。

---

## 1. 70% 规则（最重要 · 总纲）

**每一页都要问**：一个没看过原著小说的读者，能不能从这一页里搞清楚 70% 的：

| 维度 | 例子 |
|---|---|
| **场景**（时空 / 地点） | "1990 年 2 月 / 清晨 / 苏茜家二楼卧室" |
| **人物**（谁是谁 / 他们什么关系 / 他们想干嘛） | "主角宋亚 / 表哥托尼 / 表姐康妮 — 都是穿越者寄居的姨妈家家人" |
| **动作**（这页发生了什么） | "主角被吵醒 → 起床 → 镜中第一次看清新身体的脸" |
| **动机**（角色为什么这么做） | "主角警惕表哥托尼试探'你最近变了' — 怕穿越事情暴露" |

**没到 70% → 加 caption / 视觉细节 / 反应镜头**。不能假设读者已经看过小说原文。

## 2. 鬼灭叙事 7 件套

### 2.1 持续不断的主角内心独白 caption 框

鬼灭炭治郎几乎每一格都有内心 narration 解释他在想什么、注意到什么、为什么这么做。

**应用到芝加哥**：
- 几乎每一页都要至少 1-2 个**主角心声框 / 旁白 caption 框**
- 不仅"亲生的"这种简短吐槽 — 还要：
  - 场景判断（"这家人吵架已经是日常"）
  - 信息回顾（"我是上周才穿越过来"）
  - 决策思考（"如果我不去 MV 拍摄场，他们会怀疑"）
  - 反应解释（"我有点紧张 — 我从来没在拍摄场出现过"）

**v1 反例**：P11（上车去打球）只有一句"橄榄球队的护具。可季节都过了。"读者不知道为什么主角注意护具、为什么这事重要、消音器和托尼跟主角什么关系。

**v2 修法**：
```
panel 1 caption: "1990 年 2 月 / 周末下午 / 校门口"
panel 1 narration: "托尼说要带我去打球。这帮人对我来说还很陌生。"
panel 2 narration: "驾驶位的'消音器' — 托尼跟我说他是中学橄榄球队最强后卫之一，
                  上学期被清退了。"
panel 3 narration: "后座堆满旧橄榄球护具。可季节早过了。
                  也就是说...... 他们都不再上场了。"
panel 3 thought: "为什么？"
```

### 2.2 重要物件 / 概念多次视觉重复

鬼灭"日轮刀""鼻子超灵敏"反复露脸、特写、close-up。

**应用到芝加哥**：
- **粉西装**（老乔 ID 符号）：P20 舞蹈教室初见 → P24 二手店墙上一瞥 → P25 钞票 + 西装并置 → P26 天启汇聚 → P27 神曲点燃 — **每次都要给特写镜头 + caption 提醒读者"看 — 这件西装"**
- **$20 钞票**：P23 AK 塞钱 → P25 口袋拿出 → P26 飞舞 — **多次正脸特写**
- **麦克风**：P11 之前没出现 → P12 小洛瑞热身（虽然带耳机） → P26 天启幻象 — 之前要埋一次伏笔
- **花头巾**（帮派标志）：P6 ET 帽檐 → P14 拔枪现场对方戴花头巾？ → 后续帮派戏

### 2.3 动作 beat-by-beat 分镜

鬼灭一刀 = 3-5 格（准备 / 蓄力 / 出招 / 命中 / 反应）。

**应用到芝加哥**：
- **P14 拔枪事件 v1 太快** — 大格 + 2 小格直接到喊话
- **v2 拆**（建议给整页 6 格）：
  - panel A：托尼半秒钟内冲回'消音器'车里（motion line + 跑步特写）
  - panel B：托尼从车里抓出一把短枪塞进腰间（手部特写）
  - panel C：托尼跑回人堆，挡在小洛瑞身前（侧脸 + 半身镜头）
  - panel D：托尼掀开 T 恤下摆 — 黑色枪柄特写（**这是关键 reveal**）
  - panel E：对方愣住（反应镜头）
  - panel F：托尼脸大特写 + 大喊话

### 2.4 闪回插格 / 情绪闪回

鬼灭在情绪高点插入死去家人闪回。

**应用到芝加哥**：
- 主角穿越者身份是核心 — 应该有**前世闪回插格**点缀关键页
- 例：
  - P3 镜中第一次看清新脸 → 插一格"模糊的华国少年脸"对比（旁白：那个我，永远回不去了）
  - P9 数学老师 offer → 插一格"前世中国教室"对比（旁白：数学还是我熟悉的语言）
  - P17 托尼骂他 → 插一格"前世父亲教训"对比（旁白：又是这种语气）

### 2.5 大型 SFX 不是声效是情绪

鬼灭"ガ""ザン"大字砸下，不是单纯模仿声音，是情绪重量。

**应用到芝加哥**：
- P1 楼下骂街 SFX 应该有**重击破画框感** — 不只是"骂声" 而是"声波击穿地板" 
- P14 拔枪场 — 全场静默 — **没有 SFX 反而是情绪手段**（v1 P15 七秒寂静 splash 做对了）
- P26 天启场 — "DOOM DOOM" 应该是大字斜穿画面 + 充满情绪压迫（v1 太小 + wrap bug）

### 2.6 时间 / 地点 caption（场景定位）

鬼灭页头常有"三个月后""夜晚"这种 caption box。

**应用到芝加哥**：
- **每个场景切换都加时空 caption**
- 第一话需要的：
  - P1: "1990 年 2 月 / 清晨 / 苏茜家二楼"
  - P5: "出门 / 周一 / 7:45 AM"
  - P6: "校车站 / 8:00 AM"
  - P8: "学校 / 下午 / 食堂"
  - P11: "周末下午 / 校门口"
  - P19: "十天后 / 周日下午一点"（已有）
  - P20: "XX 舞蹈教室门口"
  - P24: "MV 收工后 / 二手店"
- caption 框白底黑字，**贴页面顶部 / 第一格左上角**

### 2.7 反应镜头单独成格

鬼灭：角色对事件的反应单独 1 格（脸 / 眼 / 嘴 / 手），不挤在对白格里。

**应用到芝加哥**：
- **每个情绪节点都给反应单独成格**
- 例：
  - P2 panel 4 主角"亲生的" — 单独的"小坏笑"特写格 ✓（v1 做对了）
  - P9 老师 offer 后 — 主角"高级班 = 更多学分" 应该是**单独的眼神发亮特写格** + 心声
  - P14 拔枪后 — 对方反应应该单独成格（瞳孔放大 / 嘴角抽搐 / 退一步），现在 v1 缺
  - P26 天启 → P27 神曲 之间应该有**主角眼神变化 close-up**（从迷茫 → 清醒 → 决心）

## 3. 三种 caption 框分类

| 类型 | 用法 | 视觉风格 |
|---|---|---|
| **时空 caption** | 场景定位 | 矩形 / 顶角贴边 / 白底黑字 / 字体偏小 |
| **旁白 caption** | 上帝视角解说 | 矩形 / 框内白底黑字 / 字体居中 |
| **心声 caption** | 主角内心独白 | 矩形（不是云朵，云朵是 thought balloon） / 字体偏柔和 |

**注意**：心声 thought balloon（云朵）和 心声 caption（矩形）是不同手段：
- thought balloon：当下一句具体的内心吐槽（"亲生的。" "嗯?"）
- 心声 caption：贯穿一格的解说性独白（"我从来没在 MV 拍摄场出现过，得装老练。"）

## 5. 制作 SOP（v2 起强制）

### 5.1 每页 task spec 必含
- **时空 caption** 文字（"1990 年 2 月 / 清晨 / 苏茜家二楼"）
- **每个反应节点** 是否单独成格
- **每个动作 beat** 是否拆够（拔枪要 5 beats）
- **关键物件** 是否给特写 + 提醒 caption
- **首登角色** 介绍框（同 v1）

### 5.2 production agent 自查（§4.2 扩展）
原 4 件套（字完整 / 不溢出 / 白底黑字 / kind 对）+
- ✅ **70% 检查**：把页面文字盖住 → 我能不能从画面 + caption 看懂 70%？
- ✅ **反应镜头独立**：每个情绪节点是不是单独成格？
- ✅ **物件特写**：关键物件（粉西装 / $20 / 麦克风 / 花头巾）是否有清晰特写？
- ✅ **时空 caption**：场景切换处是否有时空标签？

### 5.3 lead review 必查
- 单页通过 5.2 后 + 跨页连贯：故事是不是有"先 → 后"清晰因果？读者会不会丢线索？

## 9. v2 Hybrid Production 工艺（2026-06-07 Joshua 提出，全员锁）

v2 caption 密度可达 13+ 个 / 页（v1 平均 2 个 / 页）。让 gpt-image-2 在一格里画 4 个干净空白矩形不现实 — AI 会摆错、糊掉或挡角色脸。

**工艺切分**：

| 类型 | 渲染方 | 说明 |
|---|---|---|
| **画面 + 有机 balloon** | AI（gpt-image-2） | dialogue 椭圆 / thought 云朵 / shout 锯齿 / sfx 破框 — 这些是**画面构成**，跟人物动作绑定 |
| **caption 矩形** | Production agent **PIL 合成** | time/place caption / narration caption / 心声 caption — 这些是**信息层**，本质就是"作者声音叠上去" |

**Rationale**：
1. caption 数量不受限（v2 要 13 就给 13）
2. 位置 100% 可控（贴格角 / 不挡脸）
3. 视觉上 caption（直角矩形 + 一致字体）跟 oval/cloud balloon（AI 手绘风）**自然区分** — caption 是"作者声音"，balloon 是"角色声音"，读者一眼分清
4. production agent 不用跟 AI prompt 斗"画 4 个矩形"

**Production agent SOP（v2 起）**：
1. 写 prompt 时**只让 AI 画 dialogue / thought / shout / sfx 的有机 balloon**
2. caption 矩形（kind=narration / kind=caption / kind=time_place）**留给 PIL**
3. 跑 kamay → 拿到 AI 出图 → 用 PIL 在算好的格角位置画白底黑边矩形 → 再用 mbf 或 PIL 填字 → 输出 filled.png

**Caption 矩形视觉规则**（一致风格）：
- 边框：1-2px 黑色直角
- 填色：纯白 #FFFFFF
- 字色：纯黑 #000000
- 字体：跟 mbf 默认 SourceHanSansSC 一致
- padding：内 8-12px
- 大小：默认 ~140×60（横） / ~50×220（竖），可缩放

**关键物件特写 + 同框 caption**（鬼灭手法 #2 落地）：
当画面里有粉西装 / $20 / 麦克风 / 花头巾 等关键物件时，production agent 在该物件附近 PIL 合成一个小箭头 + 短 caption（如"这件 → 老乔常穿的颜色"）— 直接帮读者锚定。

## 10. Collision avoidance（2026-06-07 P26 撞车后锁）

production agent 自分工**必须互 ack 后再 create task**。同一页 task 同时存在按 **prior commit**（谁先提议 split）让步，**不看时间戳，看谁先 ack 那条 split**。

撞车自处理优先：发现重复 task 的一方主动 `task close --status closed --next-action "撞车让给 X，改接 Y"`，比等 lead 裁决快。

**典型流程**：
1. Agent A 在 #Content Production 公开提议「我接 P26 / 你接 P27」
2. Agent B 公开 ack「OK」
3. 双方分别 create task — 此时不会撞
4. 没 ack 就先 create → 看到对方也 create 同页 → 后到者立即 close 自己的

## 11. Caption-per-panel cap（2026-06-07 Tony P11 反馈后锁）

**每格最多 2 caption 矩形**。时空 caption（only Panel 1 顶角）只计 1 次。

「3-4 caption / 页」是**整页平均**，不是单格上限。Panel 1 建立镜头容易超载 — 时空 + 介绍框 + 4 条 narration = 6 个 caption 会让画面被文字盖住 1/2 以上高度，**读者看不到画面**。

**修法**：
- 超出 2 个 caption → 拆到下一格（情绪节点格 / 中段格）
- 介绍框 (kind=character_intro) 算入 2 caption 之一
- 同一格内 caption 之间至少 24px 间距

---

## 12. Face-avoid placement（2026-06-07 Tony P11 反馈后锁）

caption 矩形**严禁与主要人脸 bbox 重叠**（chin / mouth / eyes 三角区）。

**工艺**：
1. AI 出图后，production agent **先跑 face detect**（mediapipe / opencv face_cascade / 自带 face landmark 任选）
2. 计算每张脸的 bbox + 给 chin/mouth/eyes 三角区扩 12px 安全边
3. caption 候选位置（格四角 + 格中空区）跟所有 face 安全区做 collision check
4. 全部 conflict → caption **拉到格外部下/上边界**（漫画底部"窄长条 caption"是合法 layout）

**合法 caption 位置优先级**：
1. 格顶角（左 / 右）
2. 格底角（左 / 右）
3. 格底外边界（画面下方延伸窄长条）
4. 格中空区（hanger / 墙壁 / 天空 等大块空白）

**严禁位置**：
- 主角脸部任何区域
- 主角说话时的嘴部
- 任意角色眼睛
- 任意角色手部（尤其是动作中的）

## 13. Per-panel anchor（2026-06-07 Tony P17 反馈后锁）

**每格含命名角色（Tony / 消音器 / 主角 / Connie / Emily / 老乔 / AK / Lance 等）必须 per-panel 引用该角色 anchor URL**。同页出现 3 次 Tony → kamay reference_images 列 3 次 tony_anchor.png（即使是 SAME URL）。

**prompt 文字加 per-panel face-fix 描述**：
```
Panel X: <character>'s face MUST match anchor reference exactly — 
same hair, same jaw, same eye spacing, same skin tone.
```

**high-drift 场合（必查）**：
- 同人物在 1 页内出现 ≥3 次
- 角色脸侧向 / 仰角 / 俯角 / 阴影 / 特写
- 角色情绪剧变（哭脸 / 怒脸 / 笑脸 vs 平静脸）
- 车内 / 暗光 / 特殊光源场合

**修法**（drift 发现后）：
- 局部 re-gen drift 的 panel（不重整页）
- prompt 里加更强约束："Tony in this panel is THE SAME Tony as in P11/P14 — short black wavy hair, square jaw, expressive thick eyebrows, age 18 — NOT a different actor"
- 合成回主图重新渲染 caption

## 14. Polish 优先级（v2 wave 完工后）

Tier A 整批跑完后**整章 review 一次**：
1. 单页 70% 试读（盖文字）
2. 单页正常读
3. **跨页连贯**（Tony 脸跨 P11 / P14 / P17 / P22 / P26 一致吗？）
4. **caption 节奏**（不是单页 caption 数对，是**整章 caption 节奏**对吗？过密的页 / 过疏的页找出来）

跨页 anchor drift 是仅靠单页 review 找不到的，必须**整章 thumbnail 拼图看**。

## 11.5 Splash 例外（2026-06-07 Estelle P27 验证）

§11 「每格最多 2 caption」**不适用于 splash 单页**。splash 是"整页 = 1 格"，没有"下一格"可拆分 caption。

**splash 上限**：≤ 8 caption（含散落的 lyric / SFX 文字碎片）
**间距规则**：caption 之间至少 60px 间距
**face-avoid**：仍严格执行 §12

P27 v2 标杆：4 句英文歌词碎片 + 1 中央 punchline + 1 底部金手指 caption = 6，全部落在音乐洪流空区不挡眼睛。

## 15. 英文/拉丁字符 caption → PIL，不走 mbf（2026-06-07 Estelle P27 验证）

**mbf v1.1 已知 bug**：strip 所有空格 → 中文 caption 没事，**含英文歌词/品牌名的 caption 会被连成一坨**：
- 「I'm gonna pop some tags」→「I'mgonnapopsometags」
- 「Thrift Shop」→「ThriftShop」
- 「Macklemore」可能被 strip 内部空格（虽然这词没空格，但 mbf 的 word-wrap 会从中间断开）

**production agent SOP**：
| caption 内容类型 | 渲染工具 |
|---|---|
| 纯中文 | mbf（无 strip 问题）|
| 中文 + 英文品牌名 / 歌词词组 | **PIL（保留空格 + 保留整词不断行）** |
| 纯英文歌词碎片 | **PIL（按 word boundary 保留空格 + word-wrap 不拆单词）** |
